Receiver Operating Characteristic

The ROC curve plots the True Positive Rate (Sensitivity) against the False Positive Rate (1 - Specificity) at various threshold settings.

  • AUC (Area Under Curve) measures the model's ability to distinguish classes. Higher is better.
  • LightGBM and XGBoost both achieve extremely competitive AUC scores (>0.97).
  • The ideal model reaches the top-left corner quickly (AUC = 1.0).

Precision-Recall Curve

For highly imbalanced datasets like credit card fraud, PR curves are often more informative than ROC curves since they focus directly on the positive class (fraud).

  • Precision: Out of all predicted fraud cases, how many are actually fraud?
  • Recall: Out of all actual fraud cases, how many did the model identify?
  • Highly useful for tuning the classification threshold depending on cost metrics.

API Integration Reference docs

Integrate CCFD Radar ML classifiers directly into your payment flows via standard JSON REST endpoints.

POST /api/predict

Evaluates a credit card transaction across all three classifiers and computes SHAP feature importance vectors.

Request Headers

Header Value
Content-Type application/json
X-API-Key sentinel_dev_key_2026 (or your custom value from .env)

Payload Fields

Field Type Description
Amount float Transaction value in USD (e.g. 45.00)
Time integer Seconds elapsed since dataset start
V1 - V28 float Confidential PCA-derived attributes
Card_Number string Optional card number string for auditing
Merchant string Optional merchant name string
Example POST Query 
curl -X POST http://localhost:5000/api/predict \
  -H "Content-Type: application/json" \
  -H "X-API-Key: sentinel_dev_key_2026" \
  -d '{
    "Amount": 142.50,
    "Time": 43200,
    "Cardholder": "John Doe",
    "Merchant": "Amazon Web Services",
    "V1": -1.35, "V2": 0.42, "V3": -0.87,
    "V4": 1.05, "V5": 0.12, "V6": -0.34,
    "V7": 0.95, "V8": 0.08, "V9": -0.15,
    ...
  }'
